## 一、UI设计总体策略

### 1.1 可用技能（Claude Code Superpowers）

本项目开发环境中已配置以下前端设计相关技能：

| 技能名称 | 用途 | 调用时机 |
|---------|------|---------|
| **frontend-design** | 前端UI设计，选择独特美学方向，避免通用AI风格 | 开始任何页面设计前必须调用 |
| **design-html** | 生成高质量HTML/CSS设计稿 | 需要快速出设计原型时 |
| **design-review** | 对已完成的UI进行审查和改进建议 | 每完成一个页面后 |
| **design-shotgun** | 快速生成多套不同风格的设计方案 | 设计方向未确定时 |

### 1.2 推荐设计风格方向

| 风格 | 英文 | 适用场景 | 特点 |
|------|------|---------|------|
| 软玻璃拟态 | Soft Glassmorphism | 答题页面、结果卡片 | 毛玻璃+柔和渐变，友好轻松 |
| 编辑式/杂志 | Editorial | 结果页、人格描述 | 大字排版+留白，"权威报告"感 |
| 暖色插画风 | Warm Illustration | 首页、品牌形象 | 配合人格头像，降低心理测试严肃感 |
| 霓虹暗色 | Neon Dark | 暗色模式备选 | 年轻用户偏好，科技感 |

### 1.3 设计原则（来自竞品分析长期记忆）

1. **零门槛接入**：免注册，首页一键开始
2. **单题单页**：降低认知负担
3. **进度可视化**：剩余题量实时显示
4. **深度内容**：结果页是核心竞争力
5. **移动优先**：先设计375px宽度的移动版
6. **分享裂变**：海报+社交分享按钮
7. **隐私保护**：不强制注册，数据本地化

### 1.4 设计交付规范

- 所有页面先出移动端设计（375px），再扩展桌面端
- 触控区域最小 48×48px
- 选项按钮间距充足，防止误触
- 色彩对比度满足 WCAG AA 标准
- 支持系统级暗色模式自动切换

## 二、81型人格图像生成方案

### 2.1 需求总览

| 类别 | 数量 | 说明 |
|------|------|------|
| 传统16型角色 | 16张 | 保持现有社区成熟角色形象，色系不变 |
| 新增65型角色 | 65张 | 需全新设计，风格需与16型统一 |

### 2.2 三层生成策略

#### 🥇 首选：Midjourney v7（品质最高，无API）

**核心功能**：
- `--cref` (Character Reference)：上传角色参考图，生成新姿势/场景时保持角色身份一致
- `--sref` (Style Reference)：锁定艺术风格，所有生成图风格统一
- Seed 控制：固定种子+微调prompt，在同一设计上迭代

**工作流程**：
```
步骤1：风格定稿
  设计3-5种候选风格 → 内部投票选定1种
  生成风格参考图 → 保存为 --sref 种子

步骤2：传统16型复刻
  上传现有16型角色图 → --cref 参考 + --sref 锁风格
  → 重新生成统一风格版本

步骤3：新增65型批量生成
  为每种人格写 prompt（描述性别/气质/配色/氛围）
  → 每型生成3-5候选 → 人工筛选最优

步骤4：后期处理
  → 去底（remove.bg / Photoshop）
  → 统一尺寸（400×400px @2x）
  → WebP 压缩
```

**成本**：Midjourney Basic $10/月 ≈ 200张，2个月足够完成全部81型

**Prompt 模板示例**：
```
[人格类型描述], character portrait, flat vector illustration style,
warm color palette, friendly expression, clean background,
simple shapes, soft gradients --ar 1:1 --sref [风格参考URL] --v 7
```

#### 🥈 备选：Stable Diffusion + LoRA（可控性最强，可自建API）

**适用场景**：如果需要 100% 程序化控制 + 完美角色一致性

**工作流程**：
1. 在 AUTOMATIC1111 / ComfyUI 中安装 SD
2. 设计定稿角色后，用 10-20 张图训练 LoRA
3. 通过 LoRA 批量生成，角色一致性近乎完美
4. 可自建 API 供程序调用

**硬件需求**：RTX 3060+ (12GB+ VRAM)，一次性成本 ¥2,000-10,000

#### 🥉 快速方案：Flux 2 Pro API（有API可集成）

**适用场景**：需要程序化批量生成，不想手动操作

| 特性 | 详情 |
|------|------|
| 提供商 | Black Forest Labs |
| API 价格 | $0.03-0.06/张 |
| 画质 | 写实风格最佳，插画风格良好 |
| 集成方式 | REST API，可写脚本批量调用 |

### 2.3 推荐决策

```
预算充足 + 追求品质 → Midjourney v7 手动生成
需要API自动化 → Stable Diffusion + LoRA 自建
快速试水 + 低预算 → Flux 2 Pro API 批量生成
```

**本项目推荐路径**：先用 Midjourney v7 定稿风格和16型 → 再用 Flux 2 Pro API 批量生成65型 → 后期统一处理

## 三、色彩与品牌系统

### 3.1 推荐配色方案

**主方案：暖色专业风**

| 用途 | 色值 | 说明 |
|------|------|------|
| 主色（CTA按钮） | `#FF6B6B` 珊瑚红 | 温暖但有行动力 |
| 辅色 | `#4ECDC4` 青绿 | 理性、平衡感 |
| 中性色文字 | `#2D3436` 深灰 | 优于纯黑，减轻视觉疲劳 |
| 背景 | `#F8F9FA` 浅灰 | 干净但不刺眼 |
| 卡片 | `#FFFFFF` + 微阴影 | 层次分明 |
| E维度色 | `#FF8A80` 暖橙 | 外向=温暖外向 |
| I维度色 | `#82B1FF` 蓝紫 | 内向=沉静内敛 |
| N维度色 | `#B388FF` 紫 | 直觉=想象力 |
| S维度色 | `#69F0AE` 绿 | 实感=落地务实 |
| T维度色 | `#40C4FF` 蓝 | 思考=冷静逻辑 |
| F维度色 | `#FF80AB` 粉 | 情感=温暖共情 |
| J维度色 | `#FFD740` 金 | 判断=秩序确定 |
| P维度色 | `#84FFFF` 青 | 感知=灵活开放 |

**中性值配色（A/B/C/D）**：使用两端的混合渐变色，视觉上体现"中间过渡"

## 四、页面设计要点

### 4.1 首页

- 极简布局，中央大标题"发现真实的自己"
- 副标题"81型人格测试 · 比传统MBTI精细5倍"
- 红色/珊瑚色 CTA 按钮"开始免费测试"
- 下方简要说明：100题 / 20-30分钟 / 无需注册
- 背景：柔和渐变或微妙的几何图案

### 4.2 答题页

- 单题单页布局
- 顶部：进度条 + 题号/总题数
- 中央：题目文字（大字号，易阅读）
- 下方：5个选项按钮（间距充足，触控友好）
- 底部：上一题/下一题导航
- 支持键盘数字键 1-5 快捷选择

### 4.3 结果页

- 顶部：人格代码（大字）+ 中文名称
- 四维度可视化（每个维度三级条：E←A→I 等）
- 分段式深度内容（人格概述→维度解析→职业建议→成长空间）
- 分享按钮（生成海报/复制链接）
- 重新测试按钮

- 尺寸：400×400px @2x，压缩后 < 50KB/张
- CDN 缓存策略：1年过期
