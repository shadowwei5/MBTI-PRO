## 一、UI设计总体策略

### 1.1 可用技能（Claude Code Superpowers）

本项目开发环境中已配置以下前端设计相关技能：

| 技能名称 | 用途 | 调用时机 |
|---------|------|---------|
| **frontend-design** | 前端UI设计，选择独特美学方向，避免通用AI风格 | 开始任何页面设计前必须调用 |
| **design-html** | 生成高质量HTML/CSS设计稿 | 需要快速出设计原型时 |
| **design-review** | 对已完成的UI进行审查和改进建议 | 每完成一个页面后 |
| **design-shotgun** | 快速生成多套不同风格的设计方案 | 设计方向未确定时 |

### 1.2 推荐设计风格方向

| 风格 | 英文 | 适用场景 | 特点 |
|------|------|---------|------|
| 软玻璃拟态 | Soft Glassmorphism | 答题页面、结果卡片 | 毛玻璃+柔和渐变，友好轻松 |
| 编辑式/杂志 | Editorial | 结果页、人格描述 | 大字排版+留白，"权威报告"感 |
| 暖色插画风 | Warm Illustration | 首页、品牌形象 | 配合人格头像，降低心理测试严肃感 |
| 霓虹暗色 | Neon Dark | 暗色模式备选 | 年轻用户偏好，科技感 |

### 1.3 设计原则（来自竞品分析长期记忆）

1. **零门槛接入**：免注册，首页一键开始
2. **单题单页**：降低认知负担
3. **进度可视化**：剩余题量实时显示
4. **深度内容**：结果页是核心竞争力
5. **移动优先**：先设计375px宽度的移动版
6. **分享裂变**：海报+社交分享按钮
7. **隐私保护**：不强制注册，数据本地化

### 1.4 设计交付规范

- 所有页面先出移动端设计（375px），再扩展桌面端
- 触控区域最小 48×48px
- 选项按钮间距充足，防止误触
- 色彩对比度满足 WCAG AA 标准
- 支持系统级暗色模式自动切换

### 2.3 实际Prompt模板

**正面提示词：**
```
low poly 低多边形矢量插画，扁平化卡通风格，纯色几何色块拼接，无渐变无阴影，整体干净简约，高清矢量质感。
```

**负面提示词：**
```
写实照片，3D渲染，厚涂手绘，渐变色彩，复杂阴影，模糊失焦，人物畸形，文字错乱，模糊不清，水印，多余装饰，杂乱背景，真人肖像，笔触纹理，画面噪点，多余肢体
```

### 2.4 生成脚本

- 脚本位置：`server/scripts/generate_images.py`
- 支持 CLI 参数：`--type CODE` 单张生成 / `--batch` 批量 / `--status` 状态查看
- 状态文件：`server/generated_images/_generation_state.json`（支持断点续传）
- API端点：`https://ark.cn-beijing.volces.com/api/v3/images/generations`
- 模型：`doubao-seedream-5-0-lite-260128`

## 四、页面设计要点

### 4.1 首页

- 极简布局，中央大标题"发现真实的自己"
- 副标题"81型人格测试 · 比传统MBTI精细5倍"
- 红色/珊瑚色 CTA 按钮"开始免费测试"
- 下方简要说明：100题 / 20-30分钟 / 无需注册
- 背景：柔和渐变或微妙的几何图案

### 4.2 答题页

- 单题单页布局
- 顶部：进度条 + 题号/总题数
- 中央：题目文字（大字号，易阅读）
- 下方：5个选项按钮（间距充足，触控友好）
- 底部：上一题/下一题导航
- 支持键盘数字键 1-5 快捷选择

### 4.3 结果页

- 顶部：人格代码（大字）+ 中文名称
- 四维度可视化（每个维度三级条：E←A→I 等）
- 分段式深度内容（人格概述→维度解析→职业建议→成长空间）
- 分享按钮（生成海报/复制链接）
- 重新测试按钮

### 5.2 头像加载策略

- 首屏仅加载用户当前人格头像
- 其他80型头像懒加载或预加载（Web Worker）
- 格式：WebP（主）+ PNG（降级）
- 尺寸：400×400px @2x，压缩后 < 50KB/张
- CDN 缓存策略：1年过期
