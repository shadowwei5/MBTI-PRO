## 一、MBTI标准评分方法

### 1.1 传统MBTI评分逻辑

传统MBTI Step I 采用 **偏好清晰度指数 (Preference Clarity Index, PCI)**：

```
对每个维度：
1. 统计该维度中支持"左侧"倾向的答案数（如支持E的答案数）
2. 统计该维度中支持"右侧"倾向的答案数（如支持I的答案数）
3. 计算差值：PCI = |E答题数 - I答题数|
4. 判定：得分高的一侧为该维度的偏好方向
5. 强度：PCI值越大，偏好越清晰
```

**问题**：传统PCI不设中间区间，即使PCI=1（只差1题）也被归入某一侧。

### 1.2 MBTI-PRO评分方法

MBTI-PRO在传统PCI基础上设置三级阈值：

```
对每个维度（E/I、S/N、P/J）：
1. 25题，每题分值为 [-2, -1, 0, +1, +2]
2. 原始总分 = Σ(每题得分 × 计分方向)
3. 原始总分范围：[-50, +50]
4. 三级判定：
   ├─ 原始分 <  -17  → 负向 (I/S/P)
   ├─ 原始分 ∈ [-16, +16] → 中性 (A/B/D)
   └─ 原始分 >  +16  → 正向 (E/N/J)
```

**T/F维度特殊处理：**
```
T/F总得分 = 主观5题得分(-10~+10) + 客观20题得分(0~40)
T/F总得分范围：[-10, +50]

三级判定：
├─ 总得分 <  10  → F (情感型)
├─ 总得分 ∈ [10, 29] → C (平衡型)
└─ 总得分 >  29  → T (思考型)
```

## 二、阈值设定的理论依据

### 2.1 正态分布假设

假设人格维度在人群中呈正态分布（心理学测量的基本假设）：

```
        I/S/P    ←16%→  A/B/D  ←68%→      E/N/J
       (极端)           (中性区间)          (极端)
                 ←16%→
                 
   -50 ─────── -17 ─────── 0 ─────── +17 ─────── +50
```

- **中性区间占比约34%**（±0.67σ）
- **两端各约33%**
- 这样设计确保三级各有合理分布比例

### 2.2 标准正态分布阈值参考

| Z分数 | 累积百分比 | 对应原始分(σ=25) |
|-------|-----------|-----------------|
| ±0.5σ | 38%/62% | ±12.5 |
| ±0.67σ | 25%/75% | ±16.75 ≈ **±17** |
| ±1.0σ | 16%/84% | ±25 |

选择±17（约±0.67σ）的理由：
- 不要太窄（否则中性过多，区分度差）
- 不要太宽（否则中性过少，回到二分法）
- 约1/3的人处于中性区间，符合"中间倾向约占1/3"的预期

### 2.3 T/F维度特殊阈值的考量

T/F维度由于客观题总分为0~40（期望值约20），主观5题期望值约0，整体总分期望着约20：

| 场景 | 主观题 | 客观题 | 总分 |
|------|--------|--------|------|
| 极端F | -10 | 0 | -10 |
| 中性F | -5 | 10 | 5 |
| 中间 | 0 | 20 | 20 |
| 中性T | 5 | 30 | 35 |
| 极端T | 10 | 40 | 50 |

阈值设定建议：
- **F区 (< 10)**：客观题表现弱(+主观偏情感)
- **C区 [10, 29]**：客观题中等偏上(+主观平衡)
- **T区 (> 29)**：客观题表现强(+主观偏思考)

## 三、评分引擎伪代码

```typescript
interface Answer {
  questionId: string;
  selectedOption: string;  // 'A' | 'B' | 'C' | 'D' | 'E'
}

interface ScoreResult {
  typeCode: string;         // 'EABJ'
  dimensionScores: {
    E_I: number;            // -50 ~ +50
    S_N: number;            // -50 ~ +50
    T_F: number;            // -10 ~ +50
    P_J: number;            // -50 ~ +50
  };
  typeChars: {
    E_I: 'E' | 'A' | 'I';
    S_N: 'N' | 'B' | 'S';
    T_F: 'T' | 'C' | 'F';
    P_J: 'J' | 'D' | 'P';
  };
  typeName: string;
  confidence: number;       // 0-1 置信度
}

function calculateScore(answers: Answer[], questions: Question[]): ScoreResult {
  const scores = { E_I: 0, S_N: 0, T_F_obj: 0, T_F_sub: 0, P_J: 0 };
  
  for (const answer of answers) {
    const question = questions.find(q => q.id === answer.questionId);
    
    if (question.type === 'likert') {
      const optionValue = getOptionValue(answer.selectedOption); // -2 ~ +2
      const adjustedValue = question.direction === 'positive' 
        ? optionValue 
        : -optionValue;
      
      if (question.dimension === 'T_F') {
        scores.T_F_sub += adjustedValue;
      } else {
        scores[question.dimension] += adjustedValue;
      }
    } else if (question.type === 'objective') {
      if (answer.selectedOption === question.correctAnswer) {
        scores.T_F_obj += 2;
      }
    }
  }
  
  // 合并T/F维度
  const T_F_total = scores.T_F_sub + scores.T_F_obj;
  
  return {
    dimensionScores: {
      E_I: scores.E_I,
      S_N: scores.S_N,
      T_F: T_F_total,
      P_J: scores.P_J,
    },
    typeChars: {
      E_I: classifyE_I(scores.E_I),
      S_N: classifyS_N(scores.S_N),
      T_F: classifyT_F(T_F_total),
      P_J: classifyP_J(scores.P_J),
    }
  };
}

function classifyE_I(score: number): 'E' | 'A' | 'I' {
  if (score > 16) return 'E';
  if (score < -17) return 'I';
  return 'A';
}

function classifyS_N(score: number): 'N' | 'B' | 'S' {
  if (score > 16) return 'N';
  if (score < -17) return 'S';
  return 'B';
}

function classifyT_F(score: number): 'T' | 'C' | 'F' {
  if (score > 29) return 'T';
  if (score < 10) return 'F';
  return 'C';
}

function classifyP_J(score: number): 'J' | 'D' | 'P' {
  if (score > 16) return 'J';
  if (score < -17) return 'P';
  return 'D';
}
```

## 四、置信度计算

```
置信度 = 1 - (各维度与最近阈值的距离 / 维度总分范围)

例如：E_I得分=25，最近阈值为+17，距离=8
E_I置信度 = 1 - (8 / 33) ≈ 0.76

四个维度取平均得出总体置信度。
置信度越高，说明测试结果越明确、越稳定。
```

## 五、阈值校准计划

V1.0发布后，收集1000+测试数据后进行统计分析：

1. **分布检验**：检验四维度得分是否符合正态分布
2. **阈值优化**：基于实际分布调整阈值，使三级各占合理比例
3. **客观题区分度分析**：对20道客观题进行项目分析（难度、区分度）
4. **中性区间验证**：验证A/B/C/D分类是否比二分法有更好的用户满意度
5. **迭代校准**：每季度基于新数据微调阈值
