## 一、主流技术栈对比

### 1.1 前端框架对比

| 维度 | Vue 3 | React 18 | Svelte |
|------|-------|----------|--------|
| 学习曲线 | 低 | 中 | 低 |
| TypeScript支持 | 优秀 | 优秀 | 良好 |
| 中文社区 | **最大** | 大 | 较小 |
| 生态成熟度 | 成熟 | **最成熟** | 发展中 |
| 构建工具 | Vite(官方) | Next.js/Vite | SvelteKit |
| 移动端方案 | uni-app | Taro/RN | SvelteKit |
| 适合场景 | 中小型SPA | 复杂大型应用 | 轻量级应用 |

**推荐：Vue 3** - 中文社区资源最丰富，Vite构建速度快，Pinia状态管理简单，适合本项目规模。

### 1.2 后端框架对比

| 维度 | Express.js | Spring Boot | Django | Go/Gin |
|------|-----------|-------------|--------|--------|
| 语言 | JS/TS | Java | Python | Go |
| 开发速度 | **快** | 中 | 快 | 中 |
| 性能 | 中 | 高 | 中 | **高** |
| 生态 | 丰富 | **最丰富** | 丰富 | 发展中 |
| 学习曲线 | 低 | 高 | 低 | 中 |
| 数据库ORM | Prisma | MyBatis+/JPA | Django ORM | GORM |
| 适合场景 | API服务 | 企业级 | 快速原型 | 高并发 |

**推荐：Express.js + TypeScript** - 前后端统一语言，开发效率高，性能满足本项目需求。若后续需企业级扩展可迁移至Spring Boot。

### 1.3 数据库对比

| 维度 | MySQL | PostgreSQL | MongoDB |
|------|-------|-----------|---------|
| 数据类型 | 关系型 | 关系型 | 文档型 |
| 复杂查询 | 良好 | **优秀** | 一般 |
| JSON支持 | 8.0后支持 | **原生支持** | **原生文档** |
| 中文支持 | 良好 | 良好 | 良好 |
| 运维成本 | **低** | 中 | 低 |
| 适合场景 | 结构化数据 | 复杂分析 | 灵活Schema |

**推荐：MySQL 8.0** - 题目和人格类型数据是高度结构化的，关系型数据库最合适。国内运维成本最低。

## 五、部署方案参考

### 5.1 推荐部署栈

| 组件 | 方案 | 估算月费(CNY) |
|------|------|-------------|
| Web服务器 | 阿里云轻量服务器(2核4G) | ¥80-120 |
| 数据库 | 阿里云RDS MySQL(1核2G) | ¥60-100 |
| 缓存 | Redis(阿里云/自建) | ¥30-60 |
| CDN | 阿里云OSS + CDN | ¥10-50 |
| 域名+SSL | 阿里云/腾讯云 | ¥50-100/年 |
| **总计** | | **¥230-430/月** |

### 5.2 成本优化方案（初期）

- Web服务器 + Redis 同机部署（节省Redis服务费）
- 题库+内容以静态JSON文件管理（免数据库）
- 纯前端静态部署到OSS + CDN（零服务器成本）
- 结果计算完全在浏览器端完成

**最低成本方案：纯静态部署，仅需域名+CDN费用（约¥60-150/年）**
